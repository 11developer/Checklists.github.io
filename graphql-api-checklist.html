GraphQL API Penetration Testing Checklist

  1. Schema Discovery and Analysis: Utilize the detailed introspection query to explore the schema and document findings. Tools: InQL Scanner, GraphQL Voyager. Example introspection query provided in earlier text.
  2. Testing for Introspection: Determine if the introspection feature is enabled, which can expose sensitive data. Tools: Burp Suite, custom scripts. Attempt to query for schema details without proper authorization.
  3. Authentication and Authorization Testing: Test for broken authentication and authorization mechanisms. Tools: Burp Suite, OWASP ZAP, Postman. Example: Manipulating JWT tokens to gain unauthorized access.
  4. Input Validation and Error Handling: Perform injection attacks like SQLi or XSS through inputs and analyze error messages for data leaks. Tools: GraphQLmap, Burp Suite.
  5. Query and Mutation Testing: Test for unexpected behaviors or unauthorized data access through complex queries and mutations. Tools: Custom scripts, Burp Suite.
  6. Batch Queries and Performance Limits: Identify performance issues or denial of service vulnerabilities through nested or batched queries. Tools: Custom performance testing scripts, Burp Suite Intruder.
  7. Configuration and Security Headers: Review security configurations, headers, and secure deployment practices. Tools: Nmap, curl, SSL Labs' SSL Test. Check security headers using curl.
  8. Logging and Monitoring Evaluation: Ensure that API usage is adequately logged and monitored. Tools: ELK Stack, Splunk. Check log details for sensitive data exposure and proper masking.
  9. Third-Party Dependency Testing: Identify vulnerabilities in third-party libraries or services. Tools: OWASP Dependency Check, Retire.js.
